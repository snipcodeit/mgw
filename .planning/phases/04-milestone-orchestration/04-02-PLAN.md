Create the next.md slash command file following the MGW command pattern. The command is read-only — it does NOT modify state, run pipelines, or create worktrees.

**Frontmatter:**
```yaml
---
name: mgw:next
description: Show next unblocked issue — what to work on now, based on declared dependencies
argument-hint: ""
allowed-tools:
  - Bash
  - Read
  - AskUserQuestion
---
```

Note: minimal allowed-tools — this is a read-only command. No Write, Edit, Task, etc.

**Objective section:** Surface the next unblocked issue across the current milestone based on dependency declarations. Local-first: reads project.json for fast answer, then does a quick `gh` API check to verify the issue is still open. Offers to run `/mgw:run` after displaying the brief.

**Execution context:** Reference state.md and github.md workflows only (read-only operations).

**Process steps:**

`<step name="load_state">`: Read `.mgw/project.json`. If not found, error: "No project initialized. Run /mgw:project first." Extract `current_milestone` and milestone's issues array. Validate milestone exists.

`<step name="resolve_dependencies">`: Implement the same dependency resolution as milestone.md (Kahn's algorithm via inline python3). Use `depends_on_slugs` to build graph. Output: ordered list of issues. Also compute for each issue: what it depends on, what it unblocks (reverse lookup).

`<step name="find_unblocked">`: From the sorted list, find issues where:
- `pipeline_stage` is 'new' (not done, not failed, not in-progress)
- ALL `depends_on` issues have `pipeline_stage == 'done'`

If multiple unblocked: first by phase_number is the recommended issue, rest are alternatives.

`<step name="handle_nothing_unblocked">`: If no issues are unblocked:
- Check if ALL issues are done → "Milestone complete! Run /mgw:milestone to finalize (close milestone, create release)."
- Otherwise: build a blocking chain report. For each blocked issue, show what it's blocked by and that blocker's status:
```
No unblocked issues.

| Issue | Blocked By | Blocker Status |
|-------|-----------|----------------|
| #44 Setup tests | #42 Auth module | ◆ In progress |
| #45 Integration | #43 API layer | ✗ Failed |

Resolve #43 to unblock #45.
```

`<step name="verify_live">`: For the recommended issue, quick GitHub check:
```bash
gh issue view ${ISSUE_NUMBER} --json state,title,labels -q '{state: .state, title: .title}'
```
If issue is not OPEN: skip to next unblocked. If gh check fails (network): proceed with local data, note "(GitHub state unverified)".

`<step name="display_brief">`: Display the full brief for the recommended issue:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 MGW ► NEXT UP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Issue:       #${NUMBER} — ${TITLE}
GSD Route:   ${gsd_route}
Labels:      ${labels}
Milestone:   ${milestone_name} (${done}/${total} complete)

Description:
  ${description_first_200_chars}

Dependencies (all done):
  ✓ #${dep1} — ${dep1_title}
  ✓ #${dep2} — ${dep2_title}

Unblocks:
  ○ #${next1} — ${next1_title}
  ○ #${next2} — ${next2_title}

───────────────────────────────────────────────────────────────
```

If other unblocked alternatives exist, list them:
```
Also unblocked:
  #${ALT1} — ${alt1_title} (${alt1_gsd_route})
  #${ALT2} — ${alt2_title} (${alt2_gsd_route})
```

`<step name="offer_run">`: After displaying the brief, ask:
```
AskUserQuestion(
  header: "Ready to Start",
  question: "Run /mgw:run #${ISSUE_NUMBER} now?",
  options: [
    { label: "Yes", description: "Start the pipeline for this issue" },
    { label: "No", description: "Just viewing — I'll run it later" },
    { label: "Pick different", description: "Choose a different unblocked issue" }
  ]
)
```

If "Yes": display `/mgw:run ${ISSUE_NUMBER}` and inform user to run it (this command is read-only, it does not invoke run.md itself — it would exceed its allowed-tools).
If "Pick different": re-display with the selected alternative.
