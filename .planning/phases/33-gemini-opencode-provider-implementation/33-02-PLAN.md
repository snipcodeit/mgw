Key implementation details:

**PROVIDER_ID:** `'opencode'`

**assertAvailable():**
- Run `opencode --version` via execSync with stdio piped
- On ENOENT: print clear install message:
  "Error: opencode CLI is not installed.\n\nInstall it with:\n  npm install -g opencode-ai\n\n(or see https://opencode.ai for installation instructions)"
  then process.exit(1)
- On other non-zero exit: print "Error: opencode CLI check failed.\nEnsure opencode is installed and on your PATH." and process.exit(1)

**getCommandsDir():**
- Return `path.join(os.homedir(), '.opencode', 'commands')` (user-level commands directory)
- Require `os` from Node built-ins
- If directory does not exist, throw: `Commands directory not found at: {dir}\nRun: mgw install-commands --provider opencode\nto install MGW commands for the OpenCode provider.`

**invoke(commandFile, userPrompt, opts):**
- `const o = opts || {}`
- OpenCode invocation: use `opencode` binary with `run` subcommand as first argument
- Build args: start with `['run']`
- If commandFile provided: push `'--system-prompt'`, commandFile to args
  (opencode supports --system-prompt <file>; this is equivalent to claude's --system-prompt-file)
- If opts.model: push `'--model'`, opts.model to args
- Do NOT push --output-format json; ignore opts.json
- Push the effective user prompt (userPrompt || 'run') as the final positional argument
- opts.dryRun: print "Would invoke: opencode " + args.join(' ') and resolve { exitCode: 0, output: '' }
- Spawn `opencode` with args; stdio: inherit when not quiet, ['pipe','pipe','pipe'] when quiet
- on 'error' with ENOENT: reject with `new Error('opencode CLI not found. See https://opencode.ai for installation instructions.')`
- on 'close': resolve { exitCode: code || 0, output }

**Exports:** `module.exports = { PROVIDER_ID, assertAvailable, invoke, getCommandsDir };`

Use `'use strict';` at top. Add JSDoc matching provider-claude.cjs style. Require path, fs, os, child_process at top.
